# Thesis Integration Workflow

Use this for light thesis integration when the user needs to turn paper outputs into a thesis package. Keep the scope to manuscript organization, evidence consistency, formatting, and similarity/ethics checks.

## Thesis Role

A thesis usually evaluates:

- coherent research line
- sufficient work volume
- chapter structure
- formatting discipline
- citation completeness
- evidence consistency
- originality and similarity compliance

Do not turn this into career planning. Treat it as paper-output integration.

## Integration Pattern

```text
paper outputs -> unified research question -> chapter map -> method/evidence consistency -> format checklist -> similarity and submission readiness
```

If multiple papers exist, align them under one thesis-level question. If only one paper exists, identify which background, method, experiment, and analysis chapters need expansion.

## Chapter Map

Default structure:

- Abstract / 摘要
- Introduction
- Related Work
- Methods or System Design
- Experiments and Analysis
- Extended Discussion or Application
- Conclusion and Future Work
- References
- Appendices

For two-paper integration:

- Chapter 1: unified problem and contribution overview
- Chapter 2: shared literature and methods
- Chapter 3: paper A method/evidence
- Chapter 4: paper B method/evidence
- Chapter 5: comparison, synthesis, limitations

## Quality Checklist

Check:

- chapter titles match the contribution logic
- figures/tables are readable and consistently numbered
- all references are complete and deduplicated
- methods and experiments do not contradict paper versions
- formatting follows the institution template
- similarity check uses the required system when specified
- AI assistance, data, ethics, and code statements follow local rules

## Output

Return:

- thesis integration route
- chapter map
- source paper/output mapping
- formatting and similarity checklist
- unresolved evidence or consistency risks
