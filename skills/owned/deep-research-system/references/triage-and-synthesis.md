# Triage And Synthesis

Retrieval is not the endpoint. This reference explains how to turn a matrix into decisions.

## Triage buckets

Use at least these buckets:

- `must-read`
- `maybe`
- `background`

Optional extra buckets:

- `drop`
- `direct baseline`
- `module to steal`
- `venue narrative only`

## Triage criteria

Prefer `must-read` when a paper:

- directly addresses the current bottleneck
- is likely to change the next experiment or next phase
- introduces a module that can map onto the current code or system
- defines a benchmark or evaluation rule you will be judged against

Prefer `background` when a paper:

- is useful context but not likely to change what you build next

## Synthesis outputs

At minimum, create:

- innovation map
- curated reading list
- coverage gap register

For stronger packages, also create:

- code or system mapping
- innovation backlog
- paper triage CSV

## Decision discipline

Do not stop at “interesting”.
Every synthesis pass should answer one or more concrete questions:

- what direction looks strongest now
- what direction looks weaker now
- what source of evidence changed your confidence
- what remains uncertain
