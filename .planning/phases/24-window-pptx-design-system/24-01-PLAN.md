# Phase 24 Plan 01: Deterministic Design Registries

## Work Packages

- [ ] Add failing tests for eight exact themes, token completeness, contrast, font minima, and deterministic brand/font resolution.
- [ ] Add failing tests for 24 exact families, at least three variants each, unique IDs, normalized geometry, safe margins, capacity, and semantic-form coverage.
- [ ] Add component definitions for text, cards, KPI, images, icons, charts, tables, process, timeline, matrix, footer, and decoration.
- [ ] Add strict asset intent/manifest selection with crop-not-stretch, provenance, style consistency, and editable-native fallback.
- [ ] Quarantine all legacy template entries as `legacy_unverified` and exclude them from recommendation.
- [ ] Validate every registry, run the complete suite, obtain independent review, and close only with evidence for all five requirements.

## Exit

Phase 25 remains blocked if any semantic form is unresolved, any family has fewer than three variants, geometry crosses the normalized safe area, typography can fall below minima, contrast or font fallback is silent, an asset can stretch or lack provenance, or legacy templates can enter automatic selection.
