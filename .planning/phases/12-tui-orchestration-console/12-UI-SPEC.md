# Phase 12 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | blessed terminal UI; no shadcn; no web framework |
| Preset | not applicable |
| Component library | blessed widgets: screen, box, list, prompt; extend existing overlays only |
| Icon library | none; use existing terminal glyph constants from `src/tui/theme.ts` |
| Font | terminal monospace, inherited from user terminal; assume 1 terminal cell per ASCII char and 2 cells for wide Unicode |

Source: Phase 12 context locks a blessed TUI extension, not a replacement. Existing files provide `dashboard.ts`, `source-toggle.ts`, and `theme.ts` as the design system.

Do not initialize shadcn for this phase. This is a terminal blessed UI, not a React/Next/Vite/web app.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Conceptual equivalent: inline separators and compact status gaps; in blessed use one space between short tokens |
| sm | 8px | Conceptual equivalent: compact row padding; in blessed use two leading spaces for nested preview/report rows |
| md | 16px | Conceptual equivalent: default modal interior rhythm; in blessed use blank line between modal sections |
| lg | 24px | Conceptual equivalent: modal edge clearance; overlays remain centered at 40-70% width and 50-80% height |
| xl | 32px | Conceptual equivalent: panel separation; preserve existing bordered Hive / Cells / Nectar columns |
| 2xl | 48px | Conceptual equivalent: major report sections; use target blocks separated by a blank line |
| 3xl | 64px | Conceptual equivalent: full-screen composition; reserve for the existing top header plus three-panel dashboard |

Exceptions: Terminal layout is cell-based. Use the existing blessed geometry as the hard contract: header height 3 rows; dashboard content top 3 and height `calc(100% - 5)`; status bar height 2 rows; Hive width 20%, Cells width 50%, Nectar width 30%; centered overlays only. Do not add pixel-based CSS or web spacing.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 1 terminal cell row | regular | 1 row |
| Label | 1 terminal cell row | semibold/bold tag | 1 row |
| Heading | 1 terminal cell row | semibold/bold tag | 1 row |
| Display | 1 terminal cell row | semibold/bold tag | 1 row |

Typography contract:
- Use exactly two weights: regular text and blessed `{bold}` text.
- Do not introduce additional text sizes; terminals do not support per-widget font sizing.
- Headings are uppercase or title-case labels with `{bold}`, not larger text.
- Long skill IDs, source descriptions, eligibility reasons, and paths must truncate or wrap within the owning widget rather than forcing horizontal overflow.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | ANSI 235 / #2C2C2C Carbon | Main dashboard background, modal background, inactive panel interiors |
| Secondary (30%) | ANSI 233 / #1a1a1a Carbon Dark and ANSI 240 / #585858 Dim Gray | Guide bar background, borders, separators, archived/inactive/disabled states |
| Accent (10%) | ANSI 214 / #FFD700 Amber | Bee brand mark, focused panel border, selected row, selected checkbox, active scope/target, preview confirmation affordance |
| Destructive | ANSI 160 / #D70000 Stinger Red | Failed operation, blocked eligibility, destructive uninstall/remove warnings only |

Accent reserved for: `BEE_ASCII` brand mark, active/focused panel border, selected list item background/foreground pair, checked source/target/include state, preview modal title and confirm hint, operation category `create` or `overwrite` when needing attention, and search match highlight.

Semantic status colors:
- Success/installed/synced: use `currentColors.installed` / Pollen Green.
- Neutral available/searchable/default-disabled/skip: use Ghost White or Dim Gray with explicit reason text.
- Destructive/remove/failed/blocked: use Stinger Red.
- Archived/excluded/disabled: use Dim Gray unless explicitly blocked, then Stinger Red.

Theme contract:
- All new UI must use `currentColors` and `PALETTE` from `src/tui/theme.ts`.
- Preserve bee, monokai, and nord theme support through `getTheme()` and `THEME_NAMES`.
- Do not hard-code blessed colors outside the existing palette/theme tokens unless adding a token to `theme.ts`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Preview Sync |
| Empty state heading | No eligible skills in this scope. |
| Empty state body | Adjust source state, include skills, or switch scope to continue. |
| Error state | Operation failed: {reason}. Review the highlighted reason, adjust source/include/exclude settings, then retry. |
| Destructive confirmation | Sync remove/overwrite: Review preview. Enter=confirm sync, Esc=cancel. |

Additional required copy:
- Preview modal title: `Sync Preview`.
- Preview modal hint: `Enter confirm  Esc cancel  Up/Down scroll`.
- Final report title: `Sync Complete`.
- Source overlay title: `External Sources`.
- Source overlay hint: `space cycle  enter save  esc cancel`.
- Include/exclude overlay title: `Skill Eligibility`.
- Include/exclude overlay hint: `i include  e exclude  d default  enter save  esc cancel`.
- Scope status text in header must remain compact: `scope:global` or `scope:project`.
- Eligibility reason label in Nectar: `Eligibility: {state} — {reason}`.

Destructive actions in this phase:
1. Sync remove: stale Skillbee-managed installs may be removed. Must appear in preview before writes.
2. Sync overwrite: modified Skillbee-managed installs may be overwritten. Must appear in preview before writes.
3. Uninstall selected: keep existing uninstall report pattern; if routed through sync, it still requires preview confirmation.
4. Exclude skill: persists override YAML and may cause future removal; no filesystem write until preview-confirmed sync.
5. Disable source: persists override YAML and may cause future skips/removals; no filesystem write until preview-confirmed sync.

## Interaction Contract

### Global navigation

| Key | Contract |
|-----|----------|
| Left / Right | Cycle focus Hive -> Cells -> Nectar without changing selection |
| Up / Down | Move within focused list or scroll Nectar/modal content |
| Space | Toggle the focused selectable item where applicable |
| Enter | Open detail or confirm the currently focused modal action |
| Esc / q | Close modal overlays; `q` on base dashboard quits |
| ? / Ctrl+/ | Open command help; help must include every new Phase 12 key |

### Existing keys to preserve

| Key | Existing behavior to preserve |
|-----|-------------------------------|
| s | Toggle global/project scope and refresh scope-specific config, manifest, status, preview, and report data |
| t | Open target multi-select overlay |
| v | Toggle list/matrix Cells view |
| T | Cycle bee/monokai/nord theme and persist preference |
| / | Search skills |
| a | Toggle archived visibility |
| i | Start install/sync flow for highlighted or selected skills |
| u | Start uninstall flow for highlighted or selected skills |
| x | Archive/unarchive highlighted or selected skills |
| S | Open source controls |

### New Phase 12 keys

| Key | Behavior |
|-----|----------|
| p | Open preview for highlighted or selected skills without executing writes |
| I | Mark highlighted or selected skills as included in scoped override YAML |
| X | Mark highlighted or selected skills as excluded in scoped override YAML; do not conflict with lowercase `x` archive |
| d | Reset highlighted or selected skill include/exclude state to default |
| S | Extend existing source overlay to tri-state: enabled, default-disabled, disabled |

If implementation finds a key conflict, keep existing behavior and use the nearest mnemonic free key, then update command help and Guide text in the same change.

## Overlay Contracts

### Source tri-state overlay

- Reuse `showSourceToggle` pattern: centered blessed list, line border, shadow, Carbon background, Amber border.
- Width: 60%; height: shrink until 70%, then scroll.
- Row format: `{stateGlyph} {sourceId}  {stateLabel}  {description}`.
- State cycle on Space: `enabled -> default-disabled -> disabled -> enabled`.
- Persist on Enter to scoped override YAML only. Esc cancels without saving.
- State glyphs: enabled `✔` in Amber, default-disabled `◦` in Ghost White, disabled `✖` in Dim Gray.

### Skill include/exclude overlay

- Use centered blessed list or compact modal for selected skills.
- Row format: `[include|exclude|default] {skillId} — {reason}`.
- `I` sets include, `X` sets exclude, `d` resets default.
- Persist to scoped override YAML immediately on Enter only; Esc cancels.
- After save, refresh skill search and keep focus in Cells.

### Sync preview modal

- Must appear before any filesystem write for install/sync.
- Title: `Sync Preview`.
- Size: width 80%, height 70%, centered, scrollable.
- Top summary line: `scope:{scope} targets:{shortLabels} create:{n} overwrite:{n} remove:{n} skip:{n}`.
- Body grouping: target first, then skill rows.
- Row format: `{glyph} {operation} {skillId} — {reasonOrDestination}`.
- Operation categories required: create, overwrite, remove, skip. If Phase 11 exposes update, render as overwrite/update according to manager category but keep the preview summary explicit.
- Confirmation: Enter confirms execution; Esc/q cancels with no writes.

### Final report modal

- Title: `Sync Complete`.
- Size: width 70%, height 60%, centered, scrollable.
- Summary must include success, skipped, failed, removed, overwritten counts when provided by manager/sync data.
- Body grouping: target first, then skill rows.
- Row statuses: success, skipped, failed, removed, overwritten.
- Close with Esc/q. Do not close automatically.

## Row and Detail Contracts

### Cells list row

Required row information, in order:
1. Selection marker: selected `[x]` or existing selected glyph in Amber; unselected `[ ]` or existing unselected glyph.
2. Skill ID, highlighted for query matches.
3. Primary target install glyph.
4. Source badge when external: `src:{sourceId}` truncated if needed.
5. Eligibility badge: `eligible`, `included`, `excluded`, `default-disabled`, `global-only`, or `blocked`.
6. Archived and broken indicators, preserving existing archived Dim Gray and health red behavior.

### Cells matrix row

Required row format:
`[x] skill-id  Cx:✔ Cd:◦ Oc:— ...  eligibility:{shortReason}`

Keep using `targetShortLabel()` and `formatMatrixCell()` for target cells.

### Nectar detail

The Matrix/detail tab must add:
- Source: `{sourceId}` or `owned`.
- Source state: enabled/default-disabled/disabled.
- Override state: included/excluded/default.
- Eligibility: state plus exact resolver reason string.
- Scope impact: global/project and command-source global-only reason where applicable.
- Install matrix using existing target short labels and glyphs.
