# Phase 23 Plan 01: Declarative Deck Compiler

## Objective

Implement a pure-Python, versioned semantic compiler that lets ordinary models choose controlled intent while deterministic rules own narrative sequence, page-form ranking, capacity, rhythm, and fallback decisions.

## Work Packages

- [ ] Add failing tests for DeckPlan v1 validation and forbidden raw design instructions.
- [ ] Add 15 archetypes with stable IDs, ordered narrative sections, and scenario aliases.
- [ ] Implement typed models and strict registry loading.
- [ ] Implement semantic-to-page and chart-intent ranking with stable tie-breaking.
- [ ] Implement capacity splitting, sparse-content preservation, and continuation semantics.
- [ ] Implement cross-slide rhythm penalties and low-confidence safe defaults.
- [ ] Emit serializable top-three decision traces with rule IDs, scores, rejected candidates, confidence, and fallback reasons.
- [ ] Validate every JSON artifact and run the complete focused suite.
- [ ] Obtain independent review and close only if all five mapped requirements have evidence.

## Exit

Exit is governed by `23-VALIDATION.md`. Phase 24 remains blocked if any archetype is missing, input can smuggle raw design/code instructions, decisions are nondeterministic, capacity can silently drop content, or a trace cannot explain the selected form.
